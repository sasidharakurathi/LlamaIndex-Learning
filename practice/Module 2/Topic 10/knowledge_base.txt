### Project: VenvDrop
**Type**: personal / tool

**Description**: A pure Python, zero-dependency CLI tool for bundling massive Python environments (like torch or llama_cpp) into portable, self-verifying archives for offline, air-gapped deployment and wireless LAN transfer.

**Problem Solved**: Solves the difficulty of deploying Python projects to machines without internet access by avoiding 'copying venv' pitfalls (broken absolute paths/interpreters) and providing a wireless 'AirDrop-like' transfer method for air-gapped systems.

**Your Role**: Lead Developer & Maintainer

**Tech Stack**:
- Python 3.8+
- Python Standard Library (Zero-dependency)
- UDP Multicast (Discovery)
- HTTP/Socket (File Streaming)
- AST (Static Analysis)
- argparse
- hashlib (SHA-256)
**Architecture**: A CLI-driven utility composed of four main modules: 'Pack' (AST scanning + wheelhouse creation), 'Unpack' (integrity verification + offline venv bootstrapping), 'Send' (UDP beacon + HTTP server), and 'Receive' (multicast discovery + HTTP client).

**Features**:
- Offline Wheelhouse generation (downloads/reuses binary wheels)
- AST-based static analysis to detect hardcoded absolute paths in source code
- AirDrop-like wireless transfer via UDP multicast discovery and 4-digit PIN authentication
- SHA-256 integrity manifest generation and verification for every bundled file
- Platform metadata embedding (OS, CPU arch, Python version) with deployment guards
- Zero runtime dependencies (pure Python stdlib)
- Interactive virtual environment collision handling (Overwrite/Backup/Abort)
- Windows MAX_PATH (260 chars) safety warnings
**Challenges**:
- Enabling file discovery and transfer between air-gapped machines without a central server
- Maintaining absolute portability across different OS environments while ensuring native extensions (wheels) match the target
- Achieving complex CLI functionality without any external library dependencies (pip, requests, etc.)
**Solutions**:
- Implemented a custom UDP multicast beacon system for LAN discovery and a Python-native HTTP server for file streaming
- Embedded a 'bundle_meta.json' platform descriptor to block incompatible deployments (e.g., x86_64 to ARM)
- Utilized standard library modules like 'ast' for code analysis and 'http.server' / 'socket' for networking
**Impact**: Simplifies the distribution of heavy AI/ML Python applications in secure or remote environments where internet access is restricted or non-existent.

**Github Link**: https://github.com/sasidharakurathi/venvdrop

**Demo Link**: https://pypi.org/project/venvdrop/

**Future Improvements**:
